# Git Treasure Hunt

In this Github repository - there are 6 flags to look for. Each are in the format `flag_{flag_number}_{description}`. You'll have to use a variety of Git/Github functionality to find them. *All* solutions are findable using the web platform or via the coding interfaces.

## Hints

Flag 1: It's in the repository!

Flag 2: Commit Messages log what you've done.

Flag 3: Be careful pushing sensitive information to git - as it'll remember it even if you delete the file!

Flag 4: Creating new branches is a good way to add functionality without disrupting existing code.

Flag 5: Releases are "stable" versions of your code that can be referenced.

Flag 6: GitHub issues are a good way to request new features.
